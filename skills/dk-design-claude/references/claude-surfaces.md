# Claude Code surface kernel

Read for consequential Explore/Reimagine work, multiple input images, or when deciding how to use Claude Code subagents (the Agent tool), the CLI, or the desktop app.

## Parallel direction cell

Use at most three read-only explorers. Give each the same short brief: product, audience, primary task, primary action, real content, constraints, register, existing components/tokens, and viewport. Do not let them write files or see a preferred answer.

- **Task architect** — propose hierarchy, macro composition, responsive priority, and state coverage.
- **Art director** — propose type roles, geometry, color relationship, signature, media, and one honest tension.
- **Product critic** — challenge generic AI patterns, accessibility, density, interaction clarity, and implementation risk.

Require each response to contain: thesis, focal order, composition, type roles, color allocation, signature, responsive rule, important states, trade-off, and three anti-goals. The main agent synthesizes three complete concepts; it does not blindly equate one explorer with one final direction.

Use parallelism only when it improves a material decision. A narrow Refine or Verify task stays single-agent. Keep the main thread focused on product truth, the selected direction, source edits, and final evidence.

## Desktop app

- Prefer rendered mobile/desktop screenshots, browser inspection, Studio, and side-by-side concept boards.
- Display local images by absolute path so the user can review them in the app.
- Use image inputs for references and before/after comparisons. State what relationship to learn; never clone pixels or trade dress.
- Use image generation only for an actual production asset or image-led uncertainty, not as a substitute for rendering the real UI.

## CLI

- Inspect the same rendered images even when the interface is terminal-first.
- Summarize concepts and before/after deltas in a compact table; include absolute artifact paths.
- Prefer `dk claude context --json`, `dk verify --summary`, and narrow JSON evidence over large logs.
- Keep progress visible; do not paste multi-megabyte reports into model context.

## Single-writer rule

Only the main agent edits product source, tokens, direction, or proof configuration. Explorers and critics are read-only. Test workers may execute independent checks after the main write is complete. Taste Lock, visual baseline, debt acceptance, policy changes, and Bridge publish remain user-authorized actions even when a subagent recommends them.
